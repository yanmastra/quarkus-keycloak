# Keycloak + PostgreSQL + Nginx — Docker Compose Setup

Single-VM deployment of Keycloak with PostgreSQL backend and Nginx SSL reverse proxy.
All data is persisted in host directories (bind mounts) outside Docker.

---

## Quick Start

### 1. Configure environment

```bash
cp .env.example .env
nano .env   # Set DOMAIN, passwords, SSL_MODE
```

### 2. Generate SSL certificate

```bash
chmod +x scripts/*.sh

# Self-signed (dev/internal):
./scripts/generate-ssl.sh selfsigned your-domain.com

# Let's Encrypt (production — requires public domain + port 80 open):
./scripts/generate-ssl.sh letsencrypt your-domain.com admin@your-domain.com
```

### 3. Generate Dockerfiles

```bash
./scripts/generate-dockerfiles.sh 24.0.4
```

### 4. Customise the realm (optional)

Edit `keycloak/import/realm-export.json`:
- Change `"realm"` and `"displayName"` to your app name
- Update `"redirectUris"` and `"webOrigins"` for your client apps
- Replace `"REPLACE_WITH_STRONG_SECRET"` values with real secrets
- Update the `"frontendUrl"` attribute to match your domain

### 5. Add your logo (optional)

Place a `logo.png` (recommended: 200×60px transparent PNG) at:
```
keycloak/themes/custom-theme/login/resources/img/logo.png
```

### 6. Launch

```bash
docker compose up -d --build
```

### 7. Verify

```bash
docker compose ps
docker compose logs -f keycloak
```

Access: `https://your-domain.com`  
Admin console: `https://your-domain.com/admin`

## First Run vs Subsequent Runs

| Situation | Behaviour |
|-----------|-----------|
| First start (empty data volume) | Realm JSON is imported automatically, sentinel file written |
| Subsequent starts | Sentinel detected, import skipped — your customisations are preserved |
| Manual re-import needed | Delete `data/keycloak-data/.realm-imported` and restart |

> ⚠️ **Warning**: Deleting the sentinel file will re-import the realm JSON and may overwrite changes made through the admin UI.

## OAuth 2 Clients (pre-configured in realm)

### `web-app` — Authorization Code Flow + PKCE
For frontend/web applications.

| Setting | Value |
|---------|-------|
| Flow | Authorization Code with PKCE (S256) |
| Client Auth | Confidential (client secret) |
| Redirect URIs | Update in realm JSON |
| Token Lifetime | 5 minutes (access), 30 minutes (session) |

**Token endpoint:**
```
POST https://your-domain.com/realms/my-realm/protocol/openid-connect/token
Content-Type: application/x-www-form-urlencoded

grant_type=authorization_code&code={code}&redirect_uri={uri}
&code_verifier={verifier}&client_id=web-app&client_secret={secret}
```

### `backend-service` — Client Credentials Flow
For machine-to-machine / API-to-API calls.

```bash
curl -X POST https://your-domain.com/realms/my-realm/protocol/openid-connect/token \
  -d "grant_type=client_credentials" \
  -d "client_id=backend-service" \
  -d "client_secret=YOUR_SECRET"
```

### Discovery URL
```
https://your-domain.com/realms/my-realm/.well-known/openid-configuration
```

## Useful Commands

```bash
# View logs
docker compose logs -f keycloak
docker compose logs -f postgres
docker compose logs -f nginx

# Restart a single service
docker compose restart keycloak

# Stop everything (data preserved)
docker compose down

# Stop and remove volumes (DESTRUCTIVE)
docker compose down -v

# Open Keycloak shell
docker compose exec keycloak bash

# Export realm (after making UI changes)
docker compose exec keycloak /opt/keycloak/bin/kc.sh export \
  --dir /opt/keycloak/data/import --realm my-realm

# Backup Postgres
docker compose exec postgres pg_dump -U keycloak keycloak > backup-$(date +%Y%m%d).sql
```

- [ ] Change all default passwords in `.env`
- [ ] Replace `REPLACE_WITH_STRONG_SECRET` in realm JSON
- [ ] Use Let's Encrypt or a trusted CA certificate in production
- [ ] Restrict Keycloak admin console access (firewall / IP allowlist)
- [ ] Enable SMTP for email verification (update `smtpServer` in realm JSON)
- [ ] Review and tighten password policy in realm JSON
- [ ] Set up automated Postgres backups
- [ ] Monitor disk space on the data volume

## Theme Customisation

The `custom-theme` applies your ERP brand:
- **Primary color**: `#4B49AC`
- **Font**: Nunito (Google Fonts)
- **Border radius**: 8px

Themes are mounted directly from the host into the container — **no image rebuild required** when you make changes.

| File | Purpose |
|------|---------|
| `keycloak/themes/custom-theme/login/resources/css/login.css` | All styles |
| `keycloak/themes/custom-theme/login/login.ftl` | Login page markup |
| `keycloak/themes/custom-theme/login/login-reset-password.ftl` | Forgot password page |
| `keycloak/themes/custom-theme/login/login-update-password.ftl` | Set new password page |

**To apply a theme change:**
```bash
# 1. Edit any file under keycloak/themes/
# 2. Restart Keycloak — no rebuild needed
docker compose restart keycloak
```

The host path is controlled by `KEYCLOAK_THEMES_PATH` in your `.env` (default: `./keycloak/themes`). This maps to `/opt/keycloak/themes` inside the container, which is Keycloak's standard theme discovery path.
